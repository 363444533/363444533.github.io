react介绍

react特点

  1. 虚拟DOM + diff 算法，尽量减少与真实DOM操作

  2. 声明式编码

    在react定义函数，无需接触真实DOM和事件系统，完整最终目标（界面），通过封装API实现目标界面

  3. 组件化

    将页面想象为一颗树，树上的每一个节点，我们都可以开发出一个独立的可复用的小组件

    每个小组件都有自己独立的css，html，js，img

    可以通过搭积木（组件）方式完成整个页面

虚拟DOM是真实DOM的映射；

虚拟DOM:将真实DOM映射成树型结构的js对象来描述真实DOM结构；

树结构中的js对象包含了对应的真实DOM信息；

真实DOM和虚拟DOM的区别

  1. 虚拟DOM内容少，真实DOM有许多不需要用的内容。导致运行时间需要更多
  2. 虚拟DOM最终仍会转换为真实DOM

虚拟DOM的好处

  1. 配合diff算法，提高运行速度

diff算法

  • 找出新旧虚拟节点之间的差距，目的尽可能做到节点复用
  • 根节点对比元素相同则引用，不相同则拆除重新创建新的DOM
  • 元素类型相同，属性不同则修改属性
  • 组件相同:组件更新时，组件实例保持不见，数据变化
  • 循环中key的唯一值，用于虚拟DOM，使用Key时会基于key值得变化来进行渲染
  • 当循环中存在插入删除操作时不要以索引为key值，因为diff算法会以key值来进行对比，但每次更新后key值会发生改变，所以会发生混乱
  • 虚拟DOM比较时不会比较内容，只会比较标签属性类名等，input里的value不会进行比较

redux

npm i redux

中文网 https://cn.redux.js.org/

react配置的仓库管理，全局数据管理，用于全局组件数据通信

提供对应的模式和工具包，理解react长裤数据何时发生变化，一级如何更新

三大原则

  1. 整个应用的数据存储在一颗对象数结构中，并且这个树结构只存在与唯一一个中
  2. State 是只读数据，唯一改变state的方法就是触发action
  3. 为了描述action如何改变state，你需要编写对应的函数方法

Redux 数据流

初始启动：

  • 使用最顶层的 root reducer 函数创建 Redux store
  • store 调用一次 root reducer，并将返回值保存为它的初始 state
  • 当视图 首次渲染时，视图组件访问 Redux store 的当前 state，并使用该数据来决定要呈现的内容。同时监听 store 的更新，以便他们可以知道 state 是否已更改

更新环节：

  • 应用程序中发生了某些事情，例如用户单击按钮
  • dispatch 一个 action 到 Redux store，例如 dispatch({type: 'counter/increment'})
  • store 用之前的 state 和当前的 action 再次运行 reducer 函数，并将返回值保存为新的 state
  • store 通知所有订阅过的视图，通知它们 store 发生更新
  • 每个订阅过 store 数据的视图 组件都会检查它们需要的 state 部分是否被更新。
  • 发现数据被更新的每个组件都强制使用新数据重新渲染，紧接着更新网页
